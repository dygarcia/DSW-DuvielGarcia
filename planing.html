Presentación

Sistema de control de versiones

Un sistema de control de versiones es la gestión de los diferentes cambios realizados en un proyecto, permitiendo tener un control de los avances del proyecto realizando versiones escaladas de forma ordenada. Estos sistemas facilitan la administración y seguimiento de las versiones de cada proyecto, este procedimiento puede realizare a mano pero es altamente recomendando utilizar un sistema de control de versiones o SVC "System Version Control", una de las mas utilizadas es GIT con su plataforma GITHUB.

Contenido

GIT

Es un sistema de control de versiones distribuido que se encarga de almacenar el versionamiento del proyecto o producto en un conjunto de instantáneas de un mini sistema de archivos. Git maneja tres estados principales para organizar los archivos, estos son:

  • Confirmado (commited) indica que los datos se almacenan de manera segura en tu base de datos local.
  • Modificado (modified) indica que has realizado modificaciones sobre uno o varios de tus archivos y aun no lo has confirmado a tu base de datos local.
  • Preparado (staged) indica que has marcado uno de los archivos modificados en su versión actual para que sea enviado en la próxima confirmación.

GITHUB

Es una plataforma para el desarrollo colaborativo de software que permite el alojamiento de proyectos implementando el sistema de control de versione GIT, el código de lo proyectos se puede almacenar de dos formas:

  • Publico: Cualquiera puede ver el código, clonarlo, además puedes crear proyectos ilimitados con cierto limite de tamaño.
  • Privado Solo tu puedes ver el código del proyecto y los colaboradores que invites a formar parte del proyecto.


GITHUB permite utilizar ciertas herramientas para realizar seguimiento al proyecto, control de cambios comparando nuevos cambios con los anteriores, revisión de paquetes para su actualización, creación de ramas para implementrar nueva funcionalidad que puede ser volátil, seguimiento de problemas y una Wiki para documentar el proyecto.

GITHUB Desktop

Es una aplicación que permite mediante una interfaz grafica realizar las operaciones de control de versionamiento del proyecto, es decir, permite realizar todas operaciones de consola mediante la interfaz grafica, además de esto permite ver los cambio realizados comparar los archivos locales con los que están almacenados en al nube es decir la plataforma GITHUB, crear un repositorio, clonarlo, ver las ramificaciones del proyecto.

Para utilizar esta aplicación debes decargarla e instalarla desde su pagina oficial GITHUB Desktop, una vez termines este proceso deberás autenticarte con tus credenciales de GITHUB y listo.

Configuración de GIT

Primero en la terminal gitbash comprobamos la instalación de GIT con el siguiente comando: 

git versión

Esto mostara en al terminal la versión de git que esta instalada en nuestro equipo.

Procede a configurar tus credenciales, las que usaste para registrarte en github, usa los siguientes comandos: 

git config global user.name "tu nombre".


git config global user.email "tu_email@example.com".


Lo anterior sirve para que cuando realices un commit en el repositorio local, se almacenen con referencia a tu cuenta.

Crear un Repositorio con github desktop y hacer el primer commit

Como primer paso debes abrir la aplicación de github desktop, iniciar sesión con tus credenciales, si no posees repositorios veras un mensaje que dice "Let's get started!", donde tendrás las siguientes opciones disponibles:

  1. Crear y clonar un repositorio de tutorial.
  2. Clonar un repositorio existente desde internet.
  3. crear un repositorio nuevo.
  4. agregar un repositorio existente desde tu disco duro.
Selecciona la opción 3:

Llena los campos que te solicita con la información de tu proyecto. Que información debe llevar cada campo?

  • Name: define el nombre de tu repositorio en tu pc y en la plataforma github.
  • Description: es un campo opcional en el cual puedes brindar una breve descripción de tu proyecto.
  • Local path: establece la ruta donde se almacenarán los archivos de tu repositorio.
  • Si seleccionas la opción README: crear un archivo de confirmación inicial con el cual defines el objetivo de tu proyecto, forma de instalación, dependencias necesarias, este archivo es lo primero que sera visto por las personas que lo visiten.
  • Git ignore: permite agregar un archivo personalizado en el cual puedes incluir que archivos serán ignorados, es decir no se enviaran a tu repositorio en al nube.
  • License: permite que agregues una licencia de código abierto en tu repositorio.

Publica tu repositorio

Al momento de crear un repositorio con github desktop solo existe en tu PC y solo tu tienes acceso a el, ademas de que estas en riesgo de que si tu equipo sufre daños puedas perder tu proyecto. Para publicar tu repositorio es necesario subir los cambios a la plataforma GITHUB 1. Publica el repositorio



Te aparecerá un recuadro con las siguientes opciones:

  • El campo Name (Nombre) y Description (Descripción) se llenarán automaticamente por las opciones que ingresaste al crear el repositorio:
  • La opción keep this code private (antener este código como privado) te permite controla quien puede visualizar y clonar tu repositorio, si la seleccionas tu repositorio solo sera visible y accesible para ti.
  • Organization (Organización) te permite publicar tu repositorio en una organización a la cual pertenezcas y que hayas registrado en github.


Haz clic en el botón Publish Repository (Publicar repositorio).

Desde github desktop puedes acceder a tu repositorio en la plataforma GITHUB, en tu repositorio has click derecho y luego en la opción View on GitHub (Ver en github) lo cual abrirá una pestaña en el navegador con al url de tu proyecto en github.

Confirma los cambios.

Al momento de crear, editar o eliminar un archivo de tu proyecto se verán cambios en github desktop, es momento de subir esos cambios, en al parte inferior izquierda Changes Cambios escribe una mensaje de confirmación con información de los cambios que realizaste debe ser breve, en Description Descripción puedes ingresar una descripción mas detallada de los cambios que has realizado.

Da click en el boton Commit to master, este boton te muestra tu rama actual que es master.

Luego haz click en el boton Push origin

Esta acción envía los cambios a tu repositorio en GitHub.
