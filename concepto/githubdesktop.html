GitHub Desktop

Es una aplicación que permite mediante una interfaz grafica realizar las operaciones de control de versionamiento del proyecto, es decir, permite realizar todas operaciones de consola mediante la interfaz grafica, además de esto permite ver los cambio realizados comparar los archivos locales con los que están almacenados en al nube es decir la plataforma GITHUB, crear un repositorio, clonarlo, ver las ramificaciones del proyecto.

Instalar GitHub Desktop y autenticar tu cuenta

Puedes instalar GitHub Desktop o cualquier sistema operativo compatible. Después de instalar la app, necesitarás iniciar sesión y autenticarte en tu cuenta de GitHub o de GitHub Enterprise antes de que puedas crear y clonar un repositorio

Crear un repositorio nuevo

Si no tienes ningún repositorio asociado con GitHub Desktop, tendrás una vista de "Let's get started!", en donde podrás elegir crear y clonar un repositorio de tutorial, clonar un repositorio existente desde internet, crear un repositorio nuevo, o agregar un repositorio existente desde tu disco duro.




Crear y clonar un repositorio de tutorial
Te recomendamos que tu primer poyecto sea crear y clonar un repositorio de tutorial para practicar cómo utilizar GitHub Desktop. Haz clic en Create a tutorial repository and clone it (Crear un repositorio de tutorial y clonarlo).

Llena los campos y selecciona tus opciones preferidas.

Que información debe llevar cada campo?

  • Name: define el nombre de tu repositorio en tu pc y en la plataforma github.
  • Description: es un campo opcional en el cual puedes brindar una breve descripción de tu proyecto.
  • Local path: establece la ruta donde se almacenarán los archivos de tu repositorio.
  • Si seleccionas la opción README: crear un archivo de confirmación inicial con el cual defines el objetivo de tu proyecto, forma de instalación, dependencias necesarias, este archivo es lo primero que sera visto por las personas que lo visiten.
  • Git ignore: permite agregar un archivo personalizado en el cual puedes incluir que archivos serán ignorados, es decir no se enviaran a tu repositorio en al nube.
  • License: permite que agregues una licencia de código abierto en tu repositorio.

Publica tu repositorio

Al momento de crear un repositorio con github desktop solo existe en tu PC y solo tu tienes acceso a el, ademas de que estas en riesgo de que si tu equipo sufre daños puedas perder tu proyecto. Para publicar tu repositorio es necesario subir los cambios a la plataforma GITHUB 1. Publica el repositorio



Te aparecerá un recuadro con las siguientes opciones:

  • El campo Name (Nombre) y Description (Descripción) se llenarán automaticamente por las opciones que ingresaste al crear el repositorio:
  • La opción keep this code private (antener este código como privado) te permite controla quien puede visualizar y clonar tu repositorio, si la seleccionas tu repositorio solo sera visible y accesible para ti.
  • Organization (Organización) te permite publicar tu repositorio en una organización a la cual pertenezcas y que hayas registrado en github.


Haz clic en el botón Publish Repository (Publicar repositorio).

Desde github desktop puedes acceder a tu repositorio en la plataforma GITHUB, en tu repositorio has click derecho y luego en la opción View on GitHub (Ver en github) lo cual abrirá una pestaña en el navegador con al url de tu proyecto en github.

Confirma los cambios.

Al momento de crear, editar o eliminar un archivo de tu proyecto se verán cambios en github desktop, es momento de subir esos cambios, en al parte inferior izquierda Changes Cambios escribe una mensaje de confirmación con información de los cambios que realizaste debe ser breve, en Description Descripción puedes ingresar una descripción mas detallada de los cambios que has realizado.



Da click en el boton Commit to master, este boton te muestra tu rama actual que es master.



Luego haz click en el boton Push origin



Esta acción envía los cambios a tu repositorio en GitHub.
